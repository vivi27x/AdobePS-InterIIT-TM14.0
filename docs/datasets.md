## Dataset Sources

We used a subset of **three datasets from Kaggle**, combining them to create a diverse training set focused on weather, lighting, style, and ambience variations:

### Dataset 1: [Landscape Pictures](https://www.kaggle.com/datasets/arnaud58/landscape-pictures)

- **Source:** Kaggle - Landscape Pictures
- **Total images:** 4,318
- **Categories:**
  - Landscapes: 900 images
  - Landscapes mountain: 900 images
  - Landscapes desert: 100 images
  - Landscapes sea: 500 images
  - Landscapes beach: 500 images
  - Landscapes island: 500 images
  - Landscapes japan: 900 images

### Dataset 2: [Day-Night-City-View-Dataset](https://www.kaggle.com/datasets/sohamndeshmukh/day-night-city-view-dataset)

- **Source:** Kaggle - Day-Night-City-View-Dataset
- **Total images:** 749
- **Categories:**
  - Night images: 227 images
  - Day images: 522 images

### Dataset 3: [Weather Image Recognition](https://www.kaggle.com/datasets/jehanbhathena/weather-dataset)

- **Source:** Kaggle - Weather Image Recognition
- **Total images:** 6,862
- **Categories:** 11 weather types (dew, fog/smog, glaze, hail, lightning, rain, rainbow, rime, sandstorm, snow)

**Total combined dataset:** ~5,700 images selected from the above three datasets.

## Caption Generation

To generate high-quality captions for our training data, we used **Gemini-2.5-flash-lite** via API calls with a carefully crafted prompt:

**Prompt used:**

```
"What is the weather, ambience and lighting of the picture and strictly don't focus on where and what the place is and answer within 40-50 words?"
```

This prompt ensures that the generated captions focus on:

- **Weather conditions** (snow, rain, clear, fog, etc.)
- **Ambience** (serene, peaceful, dramatic, etc.)
- **Lighting** (soft, diffused, artificial, natural, warm, cool, etc.)

While explicitly avoiding location-specific or object-focused descriptions, aligning with our goal of semantic understanding of atmospheric and visual qualities.

## Dataset Format

`dataset.json` is a list of JSON objects, each with:

- `relative_path`: relative path to the image file (from script directory)
- `prompt`: the **user question / instruction** (same prompt used for all images)
- `caption`: the **assistant answer / target text** (generated by Gemini-2.5-flash-lite)

**Example entries:**

For example-

1) 
![Example of image we used to make our dataset](/docs//dataset_img/5061.jpg)

We added the following in the .json file-
    {
        "relative_path": "dataset\\5061.jpg",
        "prompt": "What is the weather, ambience and lighting of the picture and strictly don't focus on where and what the place is and answer within 40-50 words?",
        "response": "The weather is snowy, creating a serene and quiet ambience. The lighting is soft and diffused due to the overcast sky and falling snow, lending a peaceful, muted quality to the scene."
    }

2) ![Dataset Sample 0](/docs//dataset_img/0.jpg)

We added the following in the .json file-
    {
        "relative_path": "dataset\\0.jpg",
        "prompt": "What is the weather, ambience and lighting of the picture and strictly don't focus on where and what the place is and answer within 40-50 words?",
        "caption": "The weather appears clear and calm, with no visible signs of precipitation or strong winds. The ambience is peaceful and serene, characterized by the soft glow of city lights reflecting on the water. The lighting is predominantly artificial, creating a warm and inviting atmosphere with a touch of mystery."
    }

## Dataset Class

`SmolVLMDataset`:

- Resolves `dataset.json` relative to the script directory.
- Loads images with `PIL.Image` and converts to RGB.
- Builds a **chat-style conversation**:
  - `role: user` → image + text prompt
  - `role: assistant` → answer text (caption)

---

## Collation & Label Masking

`collate_fn`:

- Uses `AutoProcessor` to:
  - Apply the SmolVLM **chat template** via `processor.apply_chat_template(...)`
  - Tokenize text and process images together (`return_tensors="pt"`, `max_length=2048`).
- Creates `labels` from `input_ids` and **masks out the question part**:
  - Finds where the **assistant answer** starts in the rendered prompt.
  - All tokens before that index are set to `-100` (ignored by the loss).
  - Fallback: mask the first 80% of tokens if the exact position cannot be found.

This trains the model to **predict only the assistant's answer**, conditioned on both image and text prompt.
